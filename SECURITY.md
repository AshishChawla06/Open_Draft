# Security Policy

Thank you for taking the time to help improve the security of OpenDraft.  
We take security seriously and appreciate responsible disclosure of any vulnerabilities.

---

## Supported Versions

OpenDraft is under active development.  
We currently provide security updates for:

| Version | Supported |
|--------|-----------|
| Latest release (`main`) | ✔ Supported |
| Older releases | ✖ Not supported |

If you are using an older version, please update to the latest release before reporting issues.

---

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Instead, contact the maintainer privately:


use GitHub’s private security advisory feature

Please include:

- A clear description of the issue  
- Steps to reproduce  
- Potential impact  
- Any suggested fixes (optional)

We aim to acknowledge all reports within **72 hours**.

---

## Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to investigate and fix the issue  
- Avoid publicly disclosing details until a patch is released  
- Avoid testing vulnerabilities on production systems or user data  

We will credit researchers who responsibly disclose issues (unless you prefer to remain anonymous).

---

## Security Best Practices for Contributors

If you contribute to the project, please:

- Avoid committing secrets, API keys, or credentials  
- Use environment variables for sensitive data  
- Follow secure coding practices  
- Report suspicious behavior or potential vulnerabilities  

---

## Thank You

Security is a community effort.  
We appreciate your help in keeping OpenDraft safe for everyone.
